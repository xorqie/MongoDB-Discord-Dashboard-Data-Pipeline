About & Commands

Architecture overview and Discord command reference

🌸

Herald

A self-hosted Discord automation bot that monitors MongoDB for new anime episodes and seasons, then posts beautifully formatted announcements to your Discord server. Built on discord.py, FastAPI, and Motor — production-ready with a real-time dashboard.

Python 3.13 discord.py FastAPI MongoDB WebSockets
🏗️ Architecture
🐍
Python 3.13
Core runtime. Async event loop via asyncio, fully non-blocking.
🤖
discord.py
Discord gateway, prefix commands, tasks loop, event handlers.
🍃
Motor / MongoDB
Async driver for episode & series data. Zero blocking calls.
⚡
FastAPI
Dashboard REST API with type-safe endpoints and OpenAPI docs.
🔌
WebSockets
Real-time log streaming, live status ticks, and event push.
📦
JSON Persistence
posted.json, metadata, seasons — survives restarts without a DB.
⚡ Discord Commands
🧪 Testing
!testepisodes
Posts a test announcement using the latest episode from MongoDB into the episodes channel.
🛡 Moderator / Admin
!testseason
Posts a test season announcement using the latest anime series from MongoDB.
🛡 Moderator / Admin
!test_sync
Debugs sync on the most recent bot message — extracts title, episode number, checks against the database.
🛡 Moderator / Admin
🔄 Operations
!force_check
Immediately triggers a full episode check. Same as the dashboard Force Scan button.
🛡 Moderator / Admin
!sync_metadata [limit]
Scans Discord messages and syncs episode metadata.
limit — messages to scan (default: 100)
🛡 Moderator / Admin
!build_metadata [limit]
Builds metadata by matching posted IDs against MongoDB and Discord messages.
limit — messages to scan (default: 500)
🛡 Moderator / Admin
!rebuild_metadata [limit]
⚠️ Clears all posted data and rebuilds from scratch. Requires ✅ reaction to confirm.
🛡 Moderator / Admin
📊 Information
!stats
Shows bot statistics: posted count, cache size, interval, thresholds, channels, and start date.
🛡 Moderator / Admin
!view_posted [limit]
Shows last N posted episodes with metadata.
limit — number of entries (default: 10)
🛡 Moderator / Admin
!logs [lines]
Posts the last N lines of bot_events.log in Discord.
lines — number of lines (default: 20)
🛡 Moderator / Admin
!check_files
Checks status of posted.json and metadata — size, count, and mismatch warnings.
🛡 Moderator / Admin
🔧 Maintenance
!fix_json
Checks both JSON files for corruption and auto-resets metadata if corrupted.
🛡 Moderator / Admin
!clear_cache
Clears the in-memory anime cache. Bot re-fetches from MongoDB on next scan.
🛡 Moderator / Admin
!reset_posted
⚠️ Clears the entire posted episodes list and metadata. Bot re-announces all episodes after START_DATE.
🛡 Moderator / Admin

Welcome to Herald Dashboard
Your self-hosted control center for anime announcements
📊
Overview at a glance
The Overview page shows total episodes and seasons posted, a full-year activity heatmap, top anime rankings, and a live activity feed — all updated in real time via WebSocket.
🌟
Pending Seasons panel
On Overview, the Pending Seasons panel cross-references MongoDB against your announcement history to show which anime seasons haven't been posted yet — with a one-click "Post This Season" button for each.
🎮
Discord Config — live dropdowns
The Discord Config page fetches your real server's channels and roles and presents them as searchable dropdowns. Changes go live immediately — no restart needed for channel/role updates.
🎨
3-axis theme system
Settings → Theme: choose Mode (dark/light), Style (Default, Midnight, Aurora), and Accent color — including a custom color picker and shareable preset codes.
📈
Analytics, Integrity Check, Health endpoint
The Analytics page shows 12-month trends, peak days/hours, and channel comparison. Integrity Check surfaces mismatches between local files and MongoDB. /api/health returns HTTP 503 when something drops — useful for pm2/systemd auto-restart.
